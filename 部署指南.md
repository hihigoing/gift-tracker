## 3️⃣ 推送到 GitHub

```bash
# 设置默认分支为 main
git branch -M main

# 添加远程仓库（替换为你的 GitHub 用户名）
git remote add origin https://github.com/你的用户名/gift-tracker.git

# 推送
git push -u origin main
```

---

## 4️⃣ 连接 Netlify

### 方式 A：网页连接（推荐）

1. **访问 Netlify**
   ```
   https://app.netlify.com
   ```

2. **登录/注册**
   - 用 GitHub 账号登录（推荐）

3. **添加新站点**
   - 点击 "Add new site"
   - 选择 "Import an existing project"

4. **连接 GitHub**
   - 点击 "Deploy with GitHub"
   - 授权 Netlify 访问 GitHub

5. **选择仓库**
   - 找到并选择 `gift-tracker` 仓库
   - 点击 "Select repository"

6. **配置构建（重要！）**
   - **Branch to deploy**: `main`
   - **Build command**: `echo 'No build needed'`
   - **Publish directory**: `public`
   - ✅ Netlify 会自动读取 `netlify.toml`

7. **部署！**
   - 点击 "Deploy site"
   - 等待 1-2 分钟

8. **获得链接！**
   - 格式：`https://随机名称.netlify.app`
   - 可以在 "Site settings" → "Change site name" 自定义

---

### 方式 B：Netlify CLI

```bash
# 安装 Netlify CLI
```

## 🔧 配置自定义域名（可选）

1. **Netlify 设置**
   - Site settings → Domain management
   - Add custom domain

2. **输入你的域名**
   - 例如：`gift-tracker.com`

3. **配置 DNS**
   - 在域名服务商添加 CNAME 记录
   - 指向：`你的站点.netlify.app`

## ⚠️ 重要：前后端分离说明

### 当前部署的是前端（PWA）

Netlify 部署的是 **纯静态前端**，需要连接后端 API。

### 后端 API 解决方案

#### 方案 1：部署到同一服务器

修改 `public/index.html` 中的 API 地址：

```javascript
const API_BASE = 'https://你的后端域名.com';
```

#### 方案 2：使用 Netlify Functions（无服务器）

将后端改造为 Netlify Functions，但需要：
- 修改 API 为无服务器架构
- 使用 Netlify 支持的数据库（如 MongoDB Atlas）

#### 方案 3：后端部署到云服务器

- 阿里云/腾讯云 VPS
- Railway.app
- Render.com
- Heroku

---

## 🎯 推荐架构

```
前端（Netlify）          后端（云服务器）
gift-tracker.netlify.app  →  api.yourdomain.com:3000
                            ↓
                        MySQL 数据库
```

### 配置跨域（CORS）

后端 `server.js` 已配置 CORS，允许所有域名访问。

如需限制，修改：

```javascript
app.use(cors({
  origin: 'https://gift-tracker.netlify.app'
}));
```

## 📞 常见问题

**Q: Netlify 部署失败？**
A: 检查 `netlify.toml` 配置，确保 `publish = "public"`

**Q: 前端无法连接后端？**
A: 修改 `public/index.html` 中的 `API_BASE` 为后端地址

**Q: 如何回滚到旧版本？**
A: Netlify 仪表板 → Deploys → 选择历史版本 → "Publish deploy"

**Q: 部署太慢？**
A: 只修改 `public` 目录内容，Netlify 会智能缓存
